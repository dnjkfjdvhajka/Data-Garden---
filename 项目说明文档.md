# Data Garden - 感官数据花园 项目说明文档

## 概念

### 数据故事
Data Garden 是一个将24小时感官数据转化为视觉化花园的交互式数据可视化项目。通过将抽象的感官数据（听觉、视觉、触觉、环境）映射到具体的花朵形态和颜色上，创造出一个随时间变化的动态花园。

### 为何选择此主题
1. **数据的诗意表达**：感官数据本身是抽象的，通过花园的形式可以赋予其诗意和美感，让数据变得生动可感。
2. **时间的循环性**：24小时是一个自然的循环，与花园的生长周期相契合，能够直观展示数据的周期性变化。
3. **文化共鸣**：结合中国传统十二时辰文化，增加了项目的文化深度和独特性。
4. **交互的直观性**：花园是一个大家都熟悉的场景，通过点击花朵查看详情的交互方式非常直观易懂。

## 数据

### 数据来源
- **原始数据**：项目使用了`sensory_data_garden.csv`作为基础数据源
- **数据库存储**：数据被导入到MySQL数据库的`sensory_flower`表中
- **缓存机制**：使用Redis进行数据缓存，提高访问速度

### 数据结构
数据包含以下字段：
- `hour`：小时（0-24）
- `flower_name`：对应时辰的花朵名称
- `volume_db`：音量（分贝）
- `light_lux`：光照强度（勒克斯）
- `temperature_c`：温度（摄氏度）
- `humidity_percent`：湿度（百分比）
- `comfort_level`：舒适度评分
- `vibe_score`：氛围评分
- `record_date`：记录日期

### 数据收集与处理过程
1. **数据收集**：通过模拟传感器数据，收集了24小时的感官数据
2. **数据清洗**：处理缺失值和异常值
3. **数据转换**：将原始数据映射到花朵的大小、颜色和形态上
4. **数据导入**：使用`import_to_sensory_flower.py`将数据导入到数据库
5. **数据验证**：使用`verify_data.py`和`verify_database.py`验证数据的完整性

## 叙事与交互设计

### 叙事逻辑
1. **首页引入**：通过带有泼墨效果的首页，营造古风水墨氛围，引入项目主题
2. **引导页过渡**：使用卷轴展开和花朵移动的动画，引导用户了解十二时辰与花朵的对应关系
3. **花园探索**：进入花园界面后，用户可以通过环形时间轴选择不同时辰，查看对应花朵的状态和数据
4. **数据详情**：点击花朵可以查看详细的感官数据和图表分析

### 交互设计意图

#### 1. 首页泼墨动画
- **意图**：营造古风水墨氛围，为整个项目奠定文化基调，吸引用户注意力
- **实现**：使用CSS动画实现墨水扩散效果，配合《诗经》诗句增强文化感

#### 2. 引导页卷轴展开
- **意图**：通过卷轴展开的仪式感，引导用户进入项目的核心概念——十二时辰与花朵的对应关系
- **实现**：使用CSS过渡动画实现卷轴从无到有的展开效果

#### 3. 花朵拖动功能
- **意图**：让用户可以自由查看所有十二时辰对应的花朵，增强探索感
- **实现**：使用JavaScript实现鼠标和触摸拖动功能，支持自由浏览

#### 4. 花朵点击详情
- **意图**：让用户深入了解每朵花的文化背景和对应的感官数据，建立数据与文化的联系
- **实现**：点击花朵弹出模态框，显示详细的花朵信息和感官解读

#### 5. 环形时间轴
- **意图**：直观展示时间的循环性，让用户可以快速定位到感兴趣的时辰
- **实现**：使用CSS和JavaScript实现圆形时间轴，点击花朵自动切换到对应时辰

#### 6. 花朵动态绽放
- **意图**：根据数据值动态调整花朵的绽放程度，让数据变化可视化
- **实现**：根据数据值添加不同的CSS类，实现花朵的缩放、抖动和发光效果

#### 7. 水墨风格图表
- **意图**：保持整体设计风格的一致性，将数据图表融入水墨风格
- **实现**：修改Chart.js配置，使用淡墨色调和毛笔笔触效果

## 技术实现

### 前端技术
- **HTML5**：构建页面结构
- **CSS3**：实现样式和动画效果
- **JavaScript**：实现交互逻辑
- **Chart.js**：实现数据可视化图表
- **响应式设计**：适配不同设备屏幕

### 后端技术
- **Python Flask**：构建后端API
- **MySQL**：存储数据
- **Redis**：缓存数据，提高性能
- **Flask-CORS**：处理跨域请求

### 技术路径
1. **数据层**：CSV数据 → 数据库存储 → Redis缓存
2. **后端层**：Flask API → 数据查询 → 缓存处理 → 响应返回
3. **前端层**：页面加载 → 数据获取 → 数据可视化 → 交互响应

## 反思

### 最满意的部分
1. **古风水墨风格**：成功将中国传统十二时辰文化与现代数据可视化相结合，创造出独特的视觉效果
2. **流畅的动画过渡**：从首页到引导页再到花园界面的过渡动画，营造了良好的用户体验
3. **交互式数据探索**：通过环形时间轴和花朵点击功能，让用户可以直观地探索24小时的感官数据变化
4. **响应式设计**：项目在不同设备上都能良好展示，提高了用户的可访问性

### 最具挑战的部分
1. **数据与视觉的映射**：如何将抽象的感官数据转化为直观的视觉元素，需要不断调整和优化
2. **动画性能优化**：在保证动画效果的同时，确保页面加载和运行的流畅性
3. **跨设备兼容性**：确保项目在不同设备和浏览器上都能正常运行
4. **数据库与缓存的集成**：实现数据的高效存储和读取，提高系统性能

### 改进空间
1. **数据源扩展**：可以考虑使用真实的传感器数据，增加项目的实用性
2. **交互功能丰富**：可以添加更多的交互方式，如拖拽调整时间范围、数据对比等
3. **视觉效果增强**：可以进一步优化动画效果，增加更多的水墨元素和视觉细节
4. **性能优化**：可以进一步优化前端代码和后端接口，提高系统的响应速度

## 总结

Data Garden 项目成功地将抽象的感官数据转化为一个美观、交互友好的可视化花园。通过融合中国传统十二时辰文化和现代数据可视化技术，创造出了一个既有文化深度又有技术含量的项目。项目的设计理念和实现方法为数据可视化领域提供了一种新的思路，展示了如何将数据与艺术、文化相结合，创造出更加生动、直观的数据体验。
